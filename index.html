por qué AHORA es viable

60.000× cómputo vs. 100× ancho de banda → brecha de 600×

  • En 20 años de GPUs, el cómputo creció ~60.000×.
  • El ancho de banda de memoria, solo ~100×.
  • Resultado: una brecha de 600× → la inferencia local está limitada por memoria, no por cómputo.
  • De ahí la importancia de la cuantización, MoE y la VRAM. El hardware de consumo ya llegó al punto en que correr local merece la pena.

[03] // fundamentos

Qué es un LLM

Definición accesible, tipos de modelos y los formatos que lo hacen posible.

definición accesible

Un LLM = un autocompletado gigante

  • Un Large Language Model predice el siguiente token (trozo de palabra) a partir del texto anterior.
  • Entrenado con enormes cantidades de texto: «aprende» patrones del lenguaje.
  • No «razona» como un humano: calcula probabilidades. Pero a gran escala emergen capacidades (resumir, programar, traducir).
Conceptos clave: tokens · contexto (memoria de la conversación) · parámetros (tamaño del modelo: 3B, 8B, 70B…).

formatos y arquitecturas

GGUF, MoE y cuantización: por qué cabe en tu PC

GGUF

Formato único y portable de llama.cpp / Ollama / LM Studio. Empaqueta pesos + metadatos.

Cuantización Q4·Q5·Q8

Comprime los pesos (16→4-8 bits). Pesa 4-8 GB y casi no pierde calidad. Elige versiones que acaban en Q4_K_M.

MoE · Mixture of Experts

Activa solo parte de los parámetros por token (35B con 3B activos → «A3B»). Rinde como uno grande gastando como uno pequeño.

[07] // memoria externa

RAG: dale documentos a tu IA

Qué es, por qué importa, y el pipeline completo.

el problema

«El contexto es una pizarra que se borra»

  • Un LLM olvida todo al cerrar la conversación. El contexto es limitado y temporal.
  • RAG (Retrieval-Augmented Generation): memoria externa persistente. El modelo consulta tus documentos antes de responder.
  • Respuestas basadas en TUS datos (privados, actualizados) sin reentrenar el modelo.
cómo funciona

Embed → Store → Retrieve → Generate

  • Chunking — se trocean los documentos.
  • Embeddings — cada trozo → vector (significado numérico).
  • Vector DB — se guardan (Chroma, Pinecone, Qdrant).
  • Retrieve + Generate — se recuperan los trozos cercanos y se inyectan en el prompt.
Demo: subir un PDF a LM Studio / AnythingLLM y preguntar sobre su contenido — todo local.
cuándo usar cada uno

Tres formas de dar memoria a un agente

Si tu trabajo es… Mejor arquitectura
Recuperar respuestas de un corpus grande y cambiante RAG
Construir conocimiento que se acumula con el tiempo LLM Wiki (Karpathy)
Actuar de forma autónoma sin que se lo pidas Fat Skills (gBrain)
No compiten, se combinan. RAG escala a 100.000+ docs pero «no aprende»; la Wiki sintetiza; las Skills actúan.

Agentes, Skills,
MCP y Loops

el vocabulario imprescindible

4 conceptos que lo desbloquean todo

// 01

Agente

Un LLM + herramientas + un bucle que decide qué hacer.

// 02

Skill

Un workflow en Markdown (pasos + criterios de salida) que el agente sigue.

// 03

MCP

Protocolo estándar para conectar el agente a datos y herramientas externas.

// 04

Loop

Percibir → decidir → actuar → observar, hasta cumplir el objetivo.

agent skills · «proceso sobre prosa»

Workflows, no documentación

  • Proceso sobre prosa — workflows que se ejecutan, no ensayos.
  • Tablas anti-racionalización — respuestas a las excusas del modelo.
  • Verificación innegociable — terminar con evidencia concreta.
  • Divulgación progresiva — activar solo lo relevante (ahorra tokens).
  • Disciplina de alcance — tocar solo lo pedido.
addyosmani.com/blog/agent-skillsui-skills.comskills.shSkill para hacer esta presentación
model context protocol

MCP: el «USB-C» de la IA

  • Estándar abierto para conectar IA a sistemas externos: datos, herramientas y workflows.
  • «Como un puerto USB-C para la IA» — construye una vez, integra en todas partes.
  • Arquitectura: cliente (agente) ↔ servidor MCP (tools, resources, prompts).
  • Soportado por Claude, ChatGPT, VS Code, Cursor… Calendar, Figma → web, Blender → impresión 3D.
loops · agentes que trabajan solos

«Fat skills, thin harness»

  • El loop convierte un LLM en agente: repite decidir → actuar → observar.
  • El harness es el código mínimo (~200 líneas): ejecución, E/S y seguridad.
  • Las skills aportan la inteligencia del workflow.
  • Con un cron, las skills se ejecutan de forma autónoma y programada.
